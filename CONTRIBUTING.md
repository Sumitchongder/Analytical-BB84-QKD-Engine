## Ways to Contribute

You may contribute through:

- Code improvements or refactoring
- Performance optimizations (simulation or analytical backend)
- New eavesdropper strategies or noise models
- Visualization enhancements or UI improvements
- Documentation and tutorial improvements
- Bug reports and reproducibility checks

---

## Getting Started

1. Fork the repository
2. Clone your fork locally:
   ```bash
   git clone https://github.com/your-username/analytical-bb84-qkd.git
````

3. Create a new branch:

   ```bash
   git checkout -b feature/your-feature-name
   ```
4. Make your changes
5. Commit with a clear, descriptive message:

   ```bash
   git commit -m "Add: concise description of contribution"
   ```
6. Push to your fork:

   ```bash
   git push origin feature/your-feature-name
   ```
7. Open a Pull Request

---

## Contribution Guidelines

* Follow clean, modular Python coding practices
* Add docstrings and inline comments where appropriate
* Keep changes focused and well-scoped
* Ensure analytical and simulation results remain consistent
* Avoid introducing unnecessary dependencies
* Maintain reproducibility (fixed seeds, deterministic logic where possible)

---

## Scientific Integrity

When contributing:

* Clearly state assumptions and approximations
* Distinguish between simulation and analytical results
* Cite relevant prior work if introducing new methods
* Avoid overstating claims beyond what results support

---

## Review Process

Pull requests are evaluated based on:

* Technical correctness
* Code clarity and maintainability
* Scientific validity
* Alignment with project goals
* Educational and research value
